How do containerization & virtualization compare?

To better Virtualization

What is compared?

This is comparison of containerization and virtualization with a virtual machine. Running multiple virtual machinces from relatively powerful hardware is still a common sceneario when developing and deploying application. Digital workspaces both feature virtualization and containerization, and the idea is to make applications as readily available and scalable as possible for users. Traditional virtualization allows for full isolation of applications so that they run independently of each other using actual resources from the underlying infrastructure but containerization takes this idea to a new level seprating and make each container indepent and easily scalable.

Virtualization Differences:

  • Hypervisor: specialized hardware, software or firmware for operating VMs on a host machine, like a server or laptop.
  • hardware: every VM is assigned not only the essential bins/libs, but also a virtualized.
  • Guest Operating system: VM requires a full-fledged guest operating system to run. The hypervisor might be ran from the host machine's OS or a bare-metal application.

    Containerization Differences:

  • Virtualized kernels: Containers uses virtual kernels to run. Every operating system uses the same kernel to run which is a security issue as a single vunerablity could affect many contrainers.
  • Lack of Hypervisor : Contrainers do not use a hypervisor hence less overhead for the container to run though, simplifying the process
  • Flexability: Containers can be deployed relativly quickly which is very diseriable for short term applications. Containers are small, fast, and portable because unlike a virtual machine, containers do not need to include a guest OS in every instance and can, instead, simply leverage the features and resources of the host OS.
  • kubernetes: Allows for automations, deployment, management and scaling of containers.
