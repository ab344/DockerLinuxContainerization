Containerization

A type of OS Virtualization

Containerization Basics

Containerization is the idea of having a packaging a operating system that could be transferred to another host easily and ran like it was running before. The container would contain all the libraries and package to get back and up running without skipping a beat. Unlike a regular virtual machine application, a containerized application is able to run in various applications without regard of hardware. This is due to the special kernel that is shared among the applications. A containerized application runs on top of the primary operating system.

Pros of containerization

  • Containers are easy to setup as the tools are universal and very simple for a new user to deploy.
    • A container do not require a guest operating system.
    • Less overhead that has to be worried by a deployer.
    • Less licensing costs
    • Same code can be used in other systems, as one code can be used to run other same container in other environments
  • Failure of containers does not affect other parts of the system
  • Containers work with Kubernetes for automate deployment and maintenance

    • Cons of containerization

  • Containers share same kernel, meaning if any vulnerability was found, it would affect every container
